Ways to Apply JS

External JS

External JavaScript refers to JavaScript code that is stored in a separate file with a .js extension.
This file contains JavaScript functions, variables, and code logic that can be reused across multiple web pages.
By linking this external JavaScript file to an HTML document using a <script> tag with the src attribute, you can maintain cleaner and more organized code, improve code reusability, and make it easier to manage and update the JavaScript functionality of a website.
This approach also helps to separate the structure (HTML), presentation (CSS), and behavior (JavaScript) of a web page, adhering to the principle of the "separation of concerns" in web development.

Internal JS

Internal JavaScript, also known as inline JavaScript, refers to JavaScript code that is directly embedded within an HTML document.
It is placed between <script> tags within the <head> or at the end of the <body> section of an HTML file.
This allows developers to include JavaScript code directly within the HTML file itself, without the need for an external .js file.
Internal JavaScript is useful for smaller scripts or for situations where the code is specific to a particular page and doesn't need to be reused elsewhere.
While convenient for quick implementations, it's generally recommended to use external JavaScript files for larger, reusable code snippets to promote cleaner, more maintainable code.
